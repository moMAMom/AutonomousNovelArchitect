# 小説執筆自律エージェントシステム設計書：Autonomous Novel Architect (ANA)

## 1. 概要

本システムは、Pythonによる工程管理（オーケストレーション）と、LM Studio上で動作するQwen 3.5（9Bモデル）のマルチロール・プロンプティングを組み合わせ、ユーザーの初期入力（キーワード・メモ）から、設定矛盾のない長編小説を自動生成することを目的とする。

## 2. 技術スタック

- **LLM:** Qwen 3.5 9B (Context Window: 65,536 tokens)
- **Platform:** LM Studio 0.4.11 (Local API Server)
- **Language:** Python 3.12
- **Infrastructure:** AMD Ryzen 7 6800H / 32GB RAM (VRAM 16GB Shared)

## 3. システムアーキテクチャ

システムは以下の4つの機能レイヤーで構成される。

### 3.1. State Manager (状態管理層)

- **Project Bible (JSON):** 世界観、キャラクター、プロット、既出の伏線を永続的に保持。
- **Progress Tracker:** 現在の執筆フェーズ（章・シーン）を管理。

### 3.2. Orchestrator (工程管理層)

- ステップごとのロール割り当て、APIリクエストの順次実行。
- 編集者の評価に基づく「リトライ（差し戻し）」の判定ロジック。

### 3.3. Multi-Agent Layer (エージェント層)

- 同一のQwen 3.5に対し、異なるシステムプロンプトを与えることで「執筆家」「編集者」「校閲者」「整合性守護者」を擬似的に分立させる。

### 3.4. API Bridge (通信層)

- OpenAI互換APIを介したLM Studioとの通信。長大なコンテキストのパースおよびエラーハンドリング。

---

## 4. エージェント・ロール定義

| ロール名 | 役割 | 重点項目 |
| :--- | :--- | :--- |
| **執筆家 (Writer)** | 本文の生成・改訂 | 描写力、感情表現、台詞回し、リズム |
| **構成編集者 (Editor)** | ドラフトの批判的評価 | 論理的整合性、プロット遵守、テンポ |
| **校閲者 (Proofreader)** | 文章の洗練・修正 | 日本語の正確性、語彙の重複回避、文末調整 |
| **整合性守護者 (Guard)** | 設定の抽出と保存 | 新規設定のBibleへの反映、矛盾の検出 |

---

## 5. 執筆ワークフロー（フェーズ別）

### Phase 1: 概念構築 (Inception)

1. **Input:** ユーザーからの断片的なキーワード。
2. **Output:**
   - 世界観設定資料（地理、歴史、法則）。
   - 主要キャラクターシート（外見、動機、秘密）。
   - 全体プロット（起承転結のビートシート）。
3. **Storage:** 全て `Project Bible` に格納。

### Phase 2: 構造設計 (Structuring)

1. **Chapter Planning:** 各章の「目的」「重要なイベント」「次章への引き」を定義。
2. **Scene Breakdown:** 各章を複数のシーン（場面）に分割し、詳細なト書きを生成。

### Phase 3: 自律執筆サイクル (Recursive Drafting)

各章（またはシーン）ごとに以下のサイクルを回す。

1. **Drafting [Writer]:**
    - Bibleとプロットを参照し、本文を生成。
2. **Critique [Editor]:**
    - 本文を読み、改善リスト（箇条書き）を出力。
    - スコアリング（100点満点）。80点未満なら「Drafting」へ差し戻し（最大N回）。
3. **Refining [Writer]:**
    - 改善リストに基づき、本文を修正。
4. **Polishing [Proofreader]:**
    - 文体をプロレベルに磨き上げ、最終稿を作成。
5. **Archiving [Guard]:**
    - 最終稿から「新しく確定した事実」を抽出し、Bibleを自動更新。

---

## 6. コンテキスト管理戦略

Qwen 3.5の65,536トークンを有効活用するため、動的にプロンプトを構成する。

- **Fixed Memory:** 常に注入する情報（世界観の根幹、主人公の設定、全章のあらすじ）。
- **Sliding Window:** 直近2〜3話の全文。
- **Summarized Memory:** それ以前の章の要約。
- **Direct Injection:** 今から書く章に関連する特定の過去設定（Guardが抽出したタグに基づく）。

## 7. 成果物管理 (Artifacts)

```text
output/[Project_Name]/
├── project_bible.json      # 常に更新される作品の脳
├── plot/
│   ├── overall_plot.txt    # 全体図
│   └── chapter_plans/      # 章ごとの詳細設計
└── manuscript/
    ├── ch01_final.txt      # 完成原稿
    └── logs/               # 執筆過程の全ドラフトと編集者のダメ出し
```

## 8. 品質保証 (Quality Control)

- **Self-Critique:** Qwen自身に「自分の書いた文章を批判させる」ことで、AI特有の単調さを排除。
- **Consistency Check:** 執筆前にGuardロールが「この章で使う設定」をBibleから読み出し、Writerに釘を刺すことで矛盾を未然に防ぐ。

### 今後の拡張性についてのメモ

- **マルチエンド生成:** 同一のBibleから分岐プロットを作成。
- **挿絵プロンプト生成:** 章ごとの情景から、画像生成AI用のプロンプトを自動抽出。
- **文体模写:** 特定の作家の著作権フリー作品をFew-shotとして与え、文体を固定。
